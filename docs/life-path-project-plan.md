# 人生路径多智能体项目企划书

## 1. 项目名称

暂定名：LifePath Lab

副标题：

`基于多智能体的人生路径推演与反事实模拟系统`

## 2. 项目定义

这是一个 Web 项目，用多智能体模拟“人生发展中的关键节点与长期演化”。

用户可以手动输入自己的关键人生参数，例如：

- 是否保研
- 是否考研
- 是否就业
- 城市选择
- 家庭条件
- 专业方向
- 感情状态
- 健康约束
- 风险偏好

系统不会直接给一句建议，而是围绕这些参数自动扩展影响因素，生成多条未来分支路径，并推演未来 3 到 10 年的人生发展轨迹。

它既支持：

- 反事实推演：如果我选 A 而不是 B，会怎样？
- 未来预测：从现在出发，几年后的我可能处在什么位置？

## 3. 产品目标

### 3.1 核心目标

让用户看到“一个选择如何通过连锁反应改变未来”。

### 3.2 产品价值

- 帮用户做复杂决策预演。
- 帮用户理解关键变量背后的长期影响。
- 把模糊纠结变成可比较的多路径结果。
- 提供可解释、可追问、可反复试验的人生沙盘。

### 3.3 非目标

本项目不做：

- 玄学算命
- 唯一正确答案
- 心理诊断
- 投资、医疗、法律等高风险确定性建议

## 4. 产品定位

这不是一个“问答机器人”，而是一个“人生情景推演引擎”。

它更接近：

- 决策沙盘
- 人生版 simulation engine
- 个人未来路径实验室

而不是：

- 星座占卜
- 简单建议生成器

## 5. 典型使用场景

### 5.1 教育决策

- 保研 vs 考研
- 本校保研 vs 外校申请
- 读博 vs 直接工作

### 5.2 职业决策

- 大厂 vs 小厂
- 一线城市 vs 二线城市
- 稳定工作 vs 创业

### 5.3 关系与生活决策

- 是否异地
- 是否换城市
- 是否承担家庭责任优先

### 5.4 个人成长决策

- 要不要投入时间做副业
- 要不要转专业或换赛道
- 要不要延迟收益去换长期可能性

## 6. 核心产品逻辑

项目的主逻辑可以概括为：

`用户输入关键参数 -> 系统扩展影响因素 -> 形成多智能体世界 -> 沿时间推进 -> 生成多分支人生路径 -> 输出可解释报告`

这里最重要的不是“算出一个结局”，而是“看清路径如何一步步形成”。

## 7. 系统中的多 Agent 设计

建议不要只做一个“未来的我”Agent，而是搭建一组共同塑造人生路径的 Agent。

### 7.1 核心 Agent 分层

#### A. 主体层

- Self Agent  
  表示当前用户自己，持有性格、能力、目标、风险偏好、情绪韧性等参数。

- Future Self Agent  
  用于在阶段性结果形成后，对路径进行内视化解释。

#### B. 关系层

- Family Agent  
  代表家庭支持、期望、压力与资源约束。

- Partner Agent  
  代表感情关系的稳定度与决策影响。

- Friend/Social Circle Agent  
  代表社交网络、信息来源、同伴比较与机会扩散。

#### C. 机构层

- School Agent  
  代表学校环境、保研规则、导师资源、学术机会。

- Mentor Agent  
  代表导师偏好、支持力度、发展建议。

- Employer Agent  
  代表岗位、招聘门槛、晋升速度、工作强度。

- City Agent  
  代表城市机会密度、生活成本、产业环境。

#### D. 环境层

- Market Agent  
  代表行业景气度和周期变化。

- Policy Agent  
  代表政策变化、招生规则、就业政策、行业监管。

- Risk Agent  
  代表健康、家庭事件、经济波动等风险扰动。

#### E. 编排层

- Director Agent  
  负责任务编排、阶段推进、分支创建与合并。

- Branch Evaluator Agent  
  负责对每条路径做阶段评分、差异解释和分支剪枝。

- Report Agent  
  负责把整个推演过程解释给用户。

## 8. 大致需求流程

### 8.1 Step 1：用户建立人生底稿

用户先输入基础信息：

- 年龄/阶段
- 学历背景
- 专业
- 家庭条件
- 城市
- 个人偏好
- 当前困惑
- 目标时间范围

系统输出一份“基础人生画像”。

### 8.2 Step 2：定义关键参数

用户设置本次推演的关键变量，例如：

- 方案 A：保研
- 方案 B：就业
- 方案 C：考研再说

同时可设：

- 哪些参数固定
- 哪些参数可变
- 哪些事件要强制纳入

### 8.3 Step 3：系统自动扩展影响因素

系统围绕关键参数自动补全：

- 资源约束
- 时间成本
- 家庭反馈
- 导师态度
- 经济压力
- 行业环境
- 城市差异
- 感情与社交变量
- 健康与心理波动

这一步会形成一张“人生因果与关系图”。

### 8.4 Step 4：生成分支起点

系统根据用户选择生成多个起始分支：

- 主分支
- 替代分支
- 风险分支
- 乐观分支
- 保守分支

每个分支都对应一组初始世界状态。

### 8.5 Step 5：多轮时间推进

系统按时间片推进世界，例如：

- 每轮一个季度
- 每轮一个学期
- 每轮一年

每一轮中，多 Agent 共同作用，更新状态：

- 学业进展
- 职业机会
- 收入变化
- 人际关系
- 心理状态
- 生活质量
- 长期目标达成度

### 8.6 Step 6：节点延展与分叉

当关键事件发生时，系统继续扩展新节点：

- 拿到 offer
- 保研失败
- 导师更换
- 家庭变故
- 感情变化
- 城市迁移
- 健康波动

每个节点都可能触发新分支。

### 8.7 Step 7：分支比较与剪枝

随着分支变多，系统要做：

- 合并高度相似分支
- 剪掉明显无意义分支
- 保留代表性路径

避免树无限爆炸。

### 8.8 Step 8：输出报告与交互

最终系统输出：

- 路径树
- 阶段时间线
- 每条路径的关键转折点
- 风险提示
- 变量敏感度分析
- 推荐关注的现实行动项

同时允许用户继续追问：

- 为什么这条路在第三年变差？
- 如果我换城市会怎样？
- 五年后的我怎么看现在这次选择？

## 9. 前端产品模块建议

### 9.1 首页

说明项目是做什么的，并提供：

- 开始新推演
- 查看历史推演
- 查看示例案例

### 9.2 参数工作台

用户输入基础画像与关键变量。

### 9.3 影响因素图谱页

展示：

- 关键人物
- 关键机构
- 外部环境因素
- 因果关系

### 9.4 路径树页面

核心页面，展示人生节点如何一路分叉与延展。

### 9.5 路径对比页面

横向对比多条路径的：

- 学业结果
- 工作结果
- 收入趋势
- 幸福感
- 风险暴露
- 可逆性

### 9.6 Agent 访谈页面

允许用户与不同 Agent 对话：

- 未来的我
- 导师
- 家人
- 公司
- 城市机会 Agent

### 9.7 报告页面

输出完整文字报告和摘要报告。

## 10. 推演引擎的大致规则

这里不展开底层工程细节，只定义产品层的运行原则。

### 10.1 状态维度

每条人生路径至少维护以下维度：

- Education
- Career
- Finance
- Relationship
- Family
- Health
- Mental State
- Opportunity Exposure
- Optionality

### 10.2 节点生成原则

节点不能凭空出现，必须来自以下三类来源：

- 用户明确输入的关键参数
- 系统从现实语境中扩展出的影响因素
- 模拟过程中由 Agent 行为触发的新事件

### 10.3 分支生成原则

只有在“关键状态发生明显偏移”时才分支，例如：

- 路径收益差异足够大
- 风险差异足够大
- 角色关系变化足够大
- 资源结构发生转折

### 10.4 结果表达原则

结果要表达为：

- 可能性区间
- 路径类型
- 触发条件

而不是绝对判断。

## 11. 多 Agent 实施方案

建议把系统拆成下面的协作链。

### 11.1 输入理解阶段

- Intake Agent  
  解析用户输入，抽取显性参数。

- Profile Agent  
  生成用户基础画像。

- Goal Agent  
  把“纠结点”翻译成可推演目标。

输出：

- 用户画像
- 关键变量表
- 推演任务定义

### 11.2 世界扩展阶段

- Factor Expansion Agent  
  自动扩展影响因素。

- Causal Graph Agent  
  构建因果图与关系图。

- Constraint Agent  
  提取硬约束和软约束。

输出：

- 人生图谱
- 约束清单
- 关键影响因子集合

### 11.3 角色建模阶段

- Self Modeling Agent
- Family Modeling Agent
- Institution Modeling Agent
- Environment Modeling Agent

输出：

- 每类 Agent 的 profile
- 各 Agent 初始状态与权重

### 11.4 分支生成阶段

- Branch Seeder Agent  
  生成第一层分支。

- Counterfactual Agent  
  生成反事实替代路径。

- Uncertainty Agent  
  注入随机扰动与外部波动。

输出：

- 初始路径集合

### 11.5 模拟推进阶段

- Director Agent  
  控制时间推进。

- Event Generator Agent  
  生成阶段事件。

- Interaction Agent  
  处理多 Agent 相互作用。

- Branch Evaluator Agent  
  判断是否分叉、合并或剪枝。

输出：

- 节点树
- 每轮状态变化
- 分支评分

### 11.6 解释输出阶段

- Report Agent  
  生成最终报告。

- Interview Agent  
  接受用户追问。

- Recommendation Agent  
  输出“现实中该关注什么”，而不是“你必须怎么做”。

输出：

- 可读报告
- 可追问解释
- 行动提示

## 13. MVP 建议

第一版不要做太大，建议只做以下范围：

- 单用户画像
- 1 个核心纠结点
- 3 到 5 条主路径
- 5 年时间跨度
- 每年一个大节点
- 路径树可视化
- 简版报告
- 未来自我访谈

先把“保研 vs 工作”这类清晰场景跑通。

## 14. 第二阶段扩展

MVP 跑通后，再加入：

- 多轮更细粒度时间推进
- 多人关系联动
- 城市/行业环境模板
- 用户自定义权重
- 路径可信度评分
- 历史推演对比
- 团队/家庭联合推演

## 15. 风险与产品边界

### 15.1 风险

- 用户会把结果误当成命运预言。
- 用户可能被负面路径放大焦虑。
- 模型可能过度自信。

### 15.2 必须加入的产品边界

- 明确声明这是情景推演，不是现实保证。
- 对负面结论给出条件说明，不直接下判词。
- 对高风险问题加提示，不输出确定性判断。

## 16. 最终一句话企划

这个项目可以被定义为：

`一个把人生关键选择转成多智能体、可分叉、可追问、可反事实推演的未来路径实验室。`

如果 MiroFish 是“群体舆论的数字沙盘”，那你的项目就是“个体人生的数字沙盘”。
