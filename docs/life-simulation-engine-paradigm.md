# 人生推演引擎范式

## 1. 文档目的

本文定义“人生路径多智能体推演系统”的核心引擎范式。

目标不是讨论页面或具体技术栈，而是回答：

1. 这套系统的推演骨架是什么。
2. 它和 MiroFish 的关系是什么。
3. 人生项目里，节点、行为、记忆、分支应该如何设计。

## 2. 一句话定义

这套引擎可以定义为：

`把用户当前的人生处境建成一个可演化的世界模型，再让多类角色围绕关键选择持续互动，并将新结果写回世界，最终生成多条未来路径。`

## 3. 与 MiroFish 的关系

这套范式和 MiroFish 在结构上是同构的，但语义不同。

### 3.1 相同点

两者都遵循同一个闭环：

- 初始输入
- 世界建模
- Agent 互动
- 状态变化
- 记忆回写
- 下一轮推演

### 3.2 不同点

MiroFish 的延展单位是传播行为：

- 发帖
- 评论
- 转发
- 关注

人生推演的延展单位是人生节点：

- 决策
- 机会
- 结果
- 风险
- 连锁变化
- 阶段反思

所以这不是“人生版舆情模拟”，而是：

`复用 MiroFish 的推演引擎范式，重写节点语义与状态语义。`

## 4. 核心闭环

人生推演引擎的核心闭环应定义为：

`初始参数 -> 生成人生图谱 -> 角色行动 -> 状态变化 -> 回写记忆 -> 产生下一轮分支`

这个闭环是整个系统的最小运行单元。

### 4.1 初始参数

用户输入的起点信息，包括：

- 当前阶段
- 学历背景
- 专业
- 家庭条件
- 所在城市
- 个人目标
- 风险偏好
- 当前纠结点
- 规划时间范围

这些参数不是答案，而是推演起点。

### 4.2 生成人生图谱

系统根据初始参数构建一张“人生因果与关系图谱”。

这张图里至少包含：

- 人
- 机构
- 环境
- 状态
- 目标
- 约束
- 事件
- 决策选项

图谱的作用不是展示漂亮关系图，而是为后续 Agent 决策和分支生成提供上下文底座。

### 4.3 角色行动

每一轮里，不同 Agent 根据自身立场、资源和当前世界状态执行动作。

人生项目中的“行动”不是社交互动，而是：

- 做决定
- 给支持
- 施加限制
- 提供机会
- 触发风险
- 改变关系

### 4.4 状态变化

动作会改变世界状态，例如：

- 学历状态变化
- 职业状态变化
- 经济状况变化
- 心理状态变化
- 家庭关系变化
- 城市机会变化

这一步决定下一轮世界是否与上一轮不同。

### 4.5 回写记忆

每一轮产生的新事实、新状态、新倾向都要写回记忆系统。

如果不回写，系统每轮都像重新算命，路径就不会连贯。

### 4.6 下一轮分支

当状态变化达到关键阈值时，系统就从当前节点产生新分支。

因此分支不是一开始一次性写死，而是在推演中逐步长出来。

## 5. 人生图谱范式

人生图谱建议拆成六类节点。

### 5.1 主体节点

- `Self`
- `FutureSelf`
- `FamilyMember`
- `Mentor`
- `Partner`
- `Friend`

### 5.2 机构节点

- `School`
- `Lab`
- `Employer`
- `City`
- `Industry`
- `PolicySystem`

### 5.3 状态节点

- `EducationState`
- `CareerState`
- `FinanceState`
- `HealthState`
- `MentalState`
- `RelationshipState`
- `OptionalityState`

### 5.4 决策节点

- `KeepGraduateRecommendation`
- `TakeEntranceExam`
- `TakeJob`
- `ChangeCity`
- `ChangeTrack`
- `ApplyForPhD`

### 5.5 事件节点

- `OfferReceived`
- `RecommendationRejected`
- `FamilyPressureRaised`
- `AdvisorSupportChanged`
- `HealthShock`
- `IndustryDownturn`

### 5.6 目标节点

- `AcademicGrowth`
- `CareerGrowth`
- `IncomeStability`
- `GeographicFreedom`
- `RelationshipStability`
- `LongTermMeaning`

## 6. 节点类型范式

人生项目里的节点应该明确分成六种。

### 6.1 决策节点

表示用户或关键角色做出的选择。

例子：

- 要不要保研
- 要不要就业
- 要不要换城市

### 6.2 机会节点

表示世界给出的潜在正向入口。

例子：

- 导师提供名额
- 公司发来 offer
- 城市提供更高密度岗位

### 6.3 结果节点

表示某次决策或事件落地后的阶段结果。

例子：

- 成功保研
- 进入工作阶段
- 没有通过筛选

### 6.4 连锁节点

表示一个结果带来的二级和三级后果。

例子：

- 城市变化导致生活成本变化
- 工作选择影响感情稳定
- 读研选择影响收入节奏

### 6.5 风险节点

表示中断、扰动、损耗或约束。

例子：

- 家庭经济压力
- 健康波动
- 行业下行

### 6.6 反思节点

表示阶段性的解释视角，用于增强可读性与可追问性。

例子：

- 五年后的自己如何评价这次决定
- 当前路径里最大的遗憾是什么

## 7. Agent 范式

建议至少保留以下五类 Agent。

### 7.1 Self Agent

表示用户本人，持有：

- 能力
- 偏好
- 目标
- 承压能力
- 决策风格

这是主行动 Agent。

### 7.2 Relationship Agents

表示和用户有持续相互作用的人。

包括：

- 家人
- 导师
- 伴侣
- 朋友

他们主要决定：

- 支持与压力
- 信息与机会
- 约束与期待

### 7.3 Institution Agents

表示制度和组织。

包括：

- 学校
- 公司
- 城市
- 行业

他们不一定像人一样“发言”，但会以规则和机会的形式影响人生。

### 7.4 Environment Agents

表示不可控背景。

包括：

- 市场环境
- 政策环境
- 健康风险
- 家庭突发事件

它们是扰动源。

### 7.5 Director Agent

这是编排层 Agent，负责：

- 推进轮次
- 判断触发了哪些事件
- 是否生成新分支
- 是否剪枝或合并分支

它不直接扮演人生角色，而是负责“导演系统”。

## 8. 行为范式

人生推演中的行为不应照搬社交平台动作，而要定义为“改变人生状态的动作”。

建议分成五类。

### 8.1 决策行为

- 申请保研
- 备考
- 投递简历
- 接受 offer
- 放弃某选择

### 8.2 资源行为

- 提供经济支持
- 提供推荐
- 提供实习机会
- 减少支持

### 8.3 关系行为

- 导师关系增强
- 家庭压力上升
- 伴侣支持下降
- 社交圈更替

### 8.4 环境行为

- 行业变热
- 行业变冷
- 政策变化
- 城市门槛提高

### 8.5 内在行为

- 自我成长
- 焦虑积累
- 动机下降
- 风险偏好变化

## 9. 状态范式

状态不是单一分数，而是多维向量。

建议至少维护以下状态轴：

- `education`
- `career`
- `finance`
- `health`
- `mental`
- `relationship`
- `family_support`
- `social_capital`
- `optionality`
- `goal_alignment`

每个分支都必须有自己的状态快照。

## 10. 记忆范式

记忆系统建议拆成三层。

### 10.1 事实记忆

记录客观发生了什么。

例子：

- 2027 年获得保研资格
- 2028 年进入某城市工作

### 10.2 状态记忆

记录当前世界的连续状态。

例子：

- 家庭支持度下降
- 心理压力持续升高

### 10.3 倾向记忆

记录路径中的角色偏移。

例子：

- 这个分支里的你变得更保守
- 导师逐步失去信心

这三层一起决定下一轮如何演化。

## 11. 分支范式

分支不能太随意，否则树会爆炸。

建议只在以下情况下触发分支：

- 关键决策发生
- 关键机会出现
- 关键风险打断
- 核心状态差异达到阈值
- 关系结构显著变化

### 11.1 分支类型

建议至少有四类：

- 主路径分支
- 替代选择分支
- 风险扰动分支
- 乐观/保守分支

### 11.2 分支剪枝

对高度相似的分支要合并，对无信息增量的分支要剪掉。

否则系统会从“推演引擎”退化成“无限树生成器”。

## 12. 时间推进范式

MiroFish 的时间单位偏短，人生项目必须改成中长期单位。

建议三种模式：

- 月度模式
- 学期模式
- 年度模式

MVP 更建议：

- 以“季度”或“学期”为一轮
- 以“3 到 5 年”为总时长

这样既有阶段感，又不至于过细。

## 13. 推演输出范式

最终输出不应是单一结论，而应至少包含以下五部分。

### 13.1 路径树

展示每条人生路径如何分叉。

### 13.2 时间线

展示关键节点的先后顺序。

### 13.3 状态曲线

展示不同路径在：

- 收入
- 压力
- 成长
- 关系
- 机会

上的变化。

### 13.4 路径解释

说明某条路径为什么变成这样。

### 13.5 角色访谈

允许用户继续问：

- 未来的我怎么看
- 导师为什么转变态度
- 哪个节点最关键

## 14. 这套范式的优势

### 14.1 可解释

比直接问答更能解释“为什么”。

### 14.2 可反事实

天然支持“如果我选另一条路”。

### 14.3 可连续

每一轮都基于上一轮，结果更连贯。

### 14.4 可交互

用户可以在中途改参数，重新推演。

## 15. 这套范式的边界

必须明确三点：

### 15.1 它不是命运预言

它输出的是条件化路径，不是确定未来。

### 15.2 它不是单轮建议器

它更适合看连锁反应，而不是一句话建议。

### 15.3 它不是无限复杂模拟器

MVP 必须限制 Agent 数量、节点数量和分支深度。

## 16. 最终范式定义

可以把这套系统最终定义成下面这句话：

`人生推演引擎是一种以人生节点为延展单位、以多角色互动为驱动力、以记忆回写为连续性保障、以分支路径为输出形式的长期情景模拟范式。`

如果用最短的话概括：

`先把人生建成世界，再让世界把未来长出来。`
